Rapidly-Exploring Random Tree Algorithm

I created a rapidly-exploring random tree (RRT) algorithm for a Mars ice core retrieval mission within the Arcadia Planitia region. A rover would be deployed to travel from the landing location to the ice locations to retrieve the ice samples. The algorithm was completed via Matlab and served the purpose of providing a visualization of the path that the sample collection vehicle (the rover) will travel from one location to another. Another purpose of the algorithm was path and cost optimization. The algorithm attempted to calculate the optimal path by creating nodes and branches in the form of a path planning tree. It calculated the path tree by forming random nodes extending outward from the starting location the destination location i.e. from one ice location to the next. Utilizing the optimized path will allow the mission to save on costs, time, power, and will allow the rover to travel around obstacles. An important purpose of RRT algorithms is for the path to find optimized paths around obstacles. The obstacles that the rover is likely to run into when traveling from one location to another may be large rocks or boulders, crevasses, or large changes in elevation such as small or large cliffs. The program had a defined starting state (the landing ad takeoff location) and a defined goal (the ice deposits). As the rover traveled from one ice site to another, the previous goal became the new starting point. Random node generation was used to guess if its path is the optimal path otherwise it will continue making branches of nodes until it finds the path. Prior to path planning, the rover knows its starting location and the end location (the location of the ice samples). However, the rover does not know its path until the program finishes running. The planning tree utilizes the A* algorithm for path and cost optimization. A* formulates its path by utilizing its starting node to create branches of nodes in order to find the shortest path. A* may also be useful when designing paths to save costs or time. We inherently accomplish this by looking for the shortest path. The branches grow one node at a time until its termination criteria, the ice sites, are recognized. The equation below shows how cost is taken into account in A*, f(n) = g(n) + h(n) where f(n), the total estimated cost through node n, is the sum of g(n), the cost of the path from its starting node to node n, and h(n), a heuristic function, is a guess of the estimated cost of the cheapest path from node n to the goal.

The red tree depicted in the images are the attempts of finding the optimal path, but when the optimal path is found, it is highlighted in black. The black line is what the rover registers as the path it needs to travel. Once the rover understands this path, it may begin traveling to the destination. The black path is the main path of interest and is the cost efficient path taken into account in A*. One downside to the program was the time it took to run the program and the lag in each run. Because Matlab ran the RRT slowly, the node creation was not as fluid as was imagined. The code would freeze and show many nodes being added at a time instead of a single node at a time.

For this project, I learned how RRT algorithms worked in Matlab. I had used Matlab many times throughout my undergraduate career, but had never attempted an RRT algorithm. I learned robotic motion planning where the robot was the Mars rover in this case. I also learned how to apply important metrics to the Mars mission by applying the A* algorithm and figuring out how the Mars rover could save time, money, and power by using path planning.
